# 小红书封面替换

## 核心规则

这不是整图生图任务，而是模板替换任务：

```text
参考图原图
+ 新上图
+ 新标题
= 最终封面
```

默认模板：`rockflow-fintech-cover`

参考图：`assets/templates/rockflow-fintech-cover/reference.jpg`

只允许改变两处：

- 上半部分图片
- 下半部分标题

其他内容必须完全不变：ROCKFLOW logo、品牌文案、底部紫色条、分隔线、白底装饰、画布尺寸、整体版式。

## 工作流程

1. 解析用户给的标题：由 LLM 按语义手动分行。标题文本必须原样保留，只允许分行，不允许改写、提炼、扩写或补充营销词。
2. 准备上图：
   - 如果用户给了图片，直接用这张图。
   - 如果用户给了正文/主题但没给图片，必须先用 `upper_visual` prompt 调用生图模型生成一张“上半部分主视觉图”，不要生成完整封面、标题、logo 或品牌区。
3. 把上图和标题合成进参考图。
4. 检查固定区域是否被改动，像素差异必须是 `0`。

## 执行边界

必须区分三种产物：

- `upper_visual_prompt`：给生图模型用的上半部分主视觉 prompt。
- `upper_image`：由生图模型按 `upper_visual_prompt` 生成的上半部分图片。
- `final_cover`：用参考模板、`upper_image` 和手动分行标题合成的完整封面。

当用户说“生成 N 张”且没有提供上图时，默认含义是：

1. 为 N 条标题/正文分别生成 N 条 `upper_visual_prompt`。
2. 用生图模型分别生成 N 张 `upper_image`。
3. 再按需合成 N 张 `final_cover`。

不允许把以下产物当成 `upper_image`：

- 本地脚本手绘的占位图、示意图、流程图或图标组合。
- 从模板参考图裁出来的旧上图。
- 只导出的 prompt 文本。

如果当前环境没有可用的生图模型或图片 API，必须明确说明“只能先导出 prompt，不能生成上图”，并停止在 `upper_visual_prompt` 阶段；不要用程序绘图替代生图。

## 标题分行

标题是用户给定文案，不是可优化的创作文案。必须遵守：

- 短标题可原样单行；需要分行时，只允许把标题拆成 2-3 行。
- 不允许增删任何词、字母、数字或标点。
- 不允许把标题改写成更“顺”的表达。
- 不允许自动补充“值得收藏”“值得关注”“别急着下结论”等尾词。
- 分行后的 `title_line_1 + title_line_2 + title_line_3` 去掉换行后，必须等于用户原始标题去掉首尾空白后的文本。

LLM 必须自己决定标题分行，按语义和阅读节奏拆，不要把标题分行交给脚本决定。分行时优先保持完整语义单元，例如：

- 不拆开英文缩写、数字编号、产品名、人名、机构名。
- 不拆开固定短语，如 `CFO办公室`、`现金流作业`、`13周滚动现金预测`。
- 口语钩子、主体对象、结论/动作可以分在不同行。

示例：

```text
原始标题：家人们来抄CFO办公室作业啦
分行：
家人们来抄
CFO办公室
作业啦
```

如果需要生成上半部分主视觉提示词，可以用：

```bash
python3 /path/to/xhs-cover-prompt/scripts/render_prompt.py \
  --template rockflow-fintech-cover \
  --title "标题" \
  --body "正文或主题" \
  --json
```

读取输出里的：

- `upper_visual`

脚本输出里的 `title_line_1`、`title_line_2`、`title_line_3` 只能当参考或校验，不能直接作为最终分行。最终传给合成脚本的标题行，必须使用 LLM 按上述规则手动确定的分行。

如果只需要替换，不需要生成 prompt，就不要展开解释模板变量。

## 上半部分主视觉 Prompt

当用户只给标题和正文、没有提供上图时，默认先按 prompt 生成一张“财经科技主视觉图”，再用合成脚本放进模板。这个 prompt 只用于生成上半部分图片，不用于生成完整封面。

默认风格：

```text
请生成一张用于小红书财经科技封面上半部分的主视觉图，不要生成完整封面，不要出现标题文字、logo、品牌标识、真实 UI、可读数字或可读图表。

主题标题：{{标题}}

内容背景：{{正文}}

请先从标题和正文中提炼 3-5 个核心视觉对象，例如：行业资产、资金流向、风险变量、基础设施、市场指标、企业经营场景等。然后把这些对象组合成一个高端财经科技编辑视觉。

画面风格固定为：深色专业金融科技场景，真实感高级数字插画或电影感商业摄影质感，深海军蓝、青蓝色、亮紫色为主色，少量金色线条表达资金流、资本开支或价值流动。构图干净、聚焦、适合作为竖版小红书封面的上半部分主视觉，横向裁切，上方和中部有视觉重点，下方可以略暗以便衔接白色标题区。

画面应表达“复杂市场问题被结构化分析”的感觉，而不是营销海报。可以使用抽象金融图表、数据流、服务器、芯片、城市金融区、会议室、研究桌面、产业设施等元素，但所有图表都必须不可读，不能出现任何文字、数字、股票代码或真实品牌。

输出：只生成主视觉图片。
```

如果正文里有非常明确的行业对象（例如 HBM、数据中心、CFO 现金流、港股 IPO、期权波动率），可以把这些对象自然写入 prompt，但不要改变固定风格和禁用项。不要让生图模型生成标题、logo、品牌区、底部紫色条或完整海报。

## 合成命令

使用这个脚本完成模板替换：

```bash
python3 /path/to/xhs-cover-prompt/scripts/composite_rockflow_cover.py \
  --upper-image /path/to/upper-image.png \
  --title-line-1 "第一行标题" \
  --title-line-2 "第二行标题" \
  --title-line-3 "第三行标题" \
  --output /path/to/output.png \
  --json
```

脚本默认坐标已经匹配 `rockflow-fintech-cover`：

- 上图替换区域：`0,7,1080,730`
- 旧标题清理区域：`40,820,1070,1215`
- 标题位置：`842,955,1068`
- 默认字体：`assets/fonts/NotoSansCJKsc-Bold.otf`，如需替换可传 `--font /path/to/font`。

脚本输出里的 `changed_pixels_outside_allowed_boxes` 必须是 `0`。不是 `0` 就说明改到了固定区域，需要调整后重跑。

## 批量生成补充

批量生成时建议固定输出结构：

- `upper_visuals/`：保存每条标题对应的真实生图上图。
- `rowXX_标题_cover.png`：保存最终封面。
- `covers_batch_XX.zip`：只打包最终封面。
- `covers_batch_XX_contact_sheet.jpg`：用于快速肉眼检查整批效果。
- `covers_batch_XX_manifest.json`：记录标题、手动分行、prompt、上图路径、封面路径和像素校验结果。

使用内置生图工具时，生成图默认保存在 `~/.codex/generated_images/...`。如果需要放入项目输出目录，复制生成图到 `upper_visuals/`，保留原始生成图，不要移动或删除。多张连续生成时，可以先创建一个本地 marker 文件，生成后按修改时间筛选 marker 之后的新图，再按生成顺序匹配标题。

每批收尾必须校验：

- 封面数量等于标题数量。
- `upper_visuals/` 里的上图数量等于标题数量。
- 每条 `title_line_1 + title_line_2 + title_line_3` 等于原标题。
- 每张封面尺寸正确，且 `changed_pixels_outside_allowed_boxes` 全部为 `0`。
- zip 中只包含最终封面，数量等于标题数量。

如果固定区域校验失败，先定位差异区域。若差异只在标题右侧边界附近，通常是标题超出 `title-clean-box`，优先用较小字号重跑，例如从默认 `104` 调到 `96`，必要时再调到 `88`。不要扩大可变区域来掩盖标题溢出。

## 重要限制

不要把整张封面交给生图模型重画。生图模型只能用于生成上半部分图片；最终封面必须由参考图合成得到。

程序化绘图、旧目录里的上图、接近示意图的占位图只能用于排查流程，不能作为合格 `upper_image`。最终 `upper_image` 必须能追溯到用户提供的图片或真实生图输出。
